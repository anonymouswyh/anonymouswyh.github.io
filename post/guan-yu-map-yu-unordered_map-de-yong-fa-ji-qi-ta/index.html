关于map与unordered_map的用法及其他

2020-09-22 2 min read

首先在C++中使用需要：

1. #include<tr1/unordered_map>
2. using namespace std::tr1;

一：查找元素是否存在

有unordered_map<int, int> mp;  要查找x是否在map中：
1. mp.find(x)! = mp.end()
2. mp.count(x)! = 0

二：插入数据

mp[x] = 1;
又或： mp.insert(pair<int, int>(x, 1))

以上为我所了解到的unordered_map的用法，然后与map相比：
map：
优点：
1. 有序性。其元素为有序排列。
2. 红黑树。使得效率高。
缺点：
由于建立红黑树使得空间占有率高。

unordered_map:
优点：由于其内部实现了哈希表，所以加快了查找效率。
缺点：哈希表的建立会比较耗费时间。

总结：

  1. map适用于对顺序有要求的问题中；而unordered_map对于查找问题会更加高效。
  2. unordered_map占有内存更多，但同时执行效率更高。

另外关于hash的一些tips：

存在一个hash函数：index = H(key)
即根据函数与关键字key可以直接找到查找值所在位置。
而取出关键字的方法有很多：如数字分析法，平方取中法，折叠法，除留余数法。

然后关于C++STL的一些tips：

  1. size_t = unsigned int
  2. vector可以被认为是一个动态数组。
  3. v.size()取数组的大小；count(v.begin(), v.end(), x)用来统计x在v序列中出现的次数，v.count(x)同理；
  4. v.push_back()为在v后面加入一个数据，如果加入的元素大于了capacity，就重新分配一个比原来capacity大两倍的内存。
  5. string a, b; a = b.insert(4, 5 ,'x') 为a变为b在第四号位置加入5个x。
